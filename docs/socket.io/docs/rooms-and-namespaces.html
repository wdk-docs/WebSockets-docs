房间和名称空间

Socket.IO允许您 “namespace” 套接字，这实际上意味着分配不同的 endpoints 或 paths。

这是一个有用的功能，可以最大限度地减少资源(TCP连接)的数量，同时通过在通信通道之间引入分离来分离应用程序中的问题。

默认命名空间

我们调用默认命名空间 /，它是默认情况下连接到的一个 Socket.IO 客户端，默认情况下是服务器监听的。

这个命名空间由 io.sockets 或简单的 io 标识:

// the following two will emit to all the sockets connected to `/`
io.sockets.emit('hi', 'everyone');
io.emit('hi', 'everyone'); // short form

每个命名空间都会发出一个 connection 事件，它接收每个 Socket 实例作为参数

io.on('connection', function(socket){
  socket.on('disconnect', function(){ });
});

自定义命名空间

要设置自定义命名空间，可以在服务器端调用 of 函数:

const nsp = io.of('/my-namespace');
nsp.on('connection', function(socket){
  console.log('someone connected');
});
nsp.emit('hi', 'everyone!');

在客户端，您告诉Socket.IO客户端连接到该命名空间:

const socket = io('/my-namespace');

重要

The namespace is an implementation detail of the Socket.IO protocol, and is not related to the actual URL of the underlying transport, which defaults to /socket.io/….

房间

在每个命名空间中，您还可以定义套接字可以 join 和 leave 的任意通道。

加入和离开

您可以调用 join 来订阅套接字到给定的通道：

io.on('connection', function(socket){
  socket.join('some room');
});

然后在广播或发射时简单地使用 to 或 in (它们是相同的):

io.to('some room').emit('some event');

要离开一个频道，你可以像 join 一样调用 leave 。

默认房间

Socket.IO中的每个 Socket 都由一个随机的，不可思议的唯一标识符 Socket＃id 标识。为方便起见，每个套接字自动加入由此ID标识的房间。

这样可以轻松地将消息广播到其他套接字:

io.on('connection', function(socket){
  socket.on('say to someone', function(id, msg){
    socket.broadcast.to(id).emit('my message', msg);
  });
});

断开

断开连接时，插座 leave 它们自动成为所有通道，并且不需要特殊的拆卸。

从外部世界发送消息

在某些情况下，您可能希望从Socket.IO进程的上下文之外向Socket.IO命名空间/房间中的套接字发出事件。

有几种方法可以解决这个问题，比如实现自己的通道将消息发送到进程中。

为了方便这个用例，我们创建了两个模块:

  • socket.io-redis

  • socket.io-emitter

通过实现Redis Adapter:

const io = require('socket.io')(3000);
const redis = require('socket.io-redis');
io.adapter(redis({ host: 'localhost', port: 6379 }));

然后，您可以从任何其他进程 emit 消息到任何通道

const io = require('socket.io-emitter')({ host: '127.0.0.1', port: 6379 });
setInterval(function(){
  io.emit('time', new Date);
}, 5000);
