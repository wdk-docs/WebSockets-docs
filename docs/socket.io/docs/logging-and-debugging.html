记录和调试¶

Socket.IO现在完全由一个名为 debug 的简约但功能强大的实用工具配备,由TJ Holowaychuk提供.

在1.0之前,Socket.IO服务器将默认将所有内容记录到控制台.对于许多用户来说这很烦人(尽管对其他用户非常有用),所以现在我们默认默认情况下是完全无声的.

基本思想是Socket.IO使用的每个模块都提供不同的调试范围,使您可以深入了解内部.默认情况下,所有输出都被抑制,您可以通过提供 DEBUG env变量(Node.JS)或 localStorage.debug 属性(浏览器)来选择查看消息.

可用的调试范围¶

查看可用信息的最佳方法是使用 *：

DEBUG=* node yourfile.js

或在浏览器中：

localStorage.debug = '*';

然后按照你感兴趣的范围进行过滤.你可以在 * 前加上范围,如果有多个则用逗号分隔.例如,要仅从Node.js上的socket.io客户端查看调试语句,请尝试以下操作：

DEBUG=socket.io:client* node yourfile.js

要查看来自引擎 和 socket.io的所有调试消息：

DEBUG=engine,socket.io* node yourfile.js
