Manager

可用选项:

选项

默认值

描述

path

/socket.io

在服务器端捕获的路径的名称

reconnection

true

是否自动重新连接

reconnectionAttempts

Infinity

放弃之前的重新连接尝试次数

reconnectionDelay

1000

在尝试重新连接之前最初等待多长时间(1000).受+/- randomizationFactor 的影响,例如默认的初始延迟将在500到1500ms之间.

reconnectionDelayMax

5000

重新连接之间等待的最长时间(5000).如上所述,每次尝试都将重新连接延迟增加2倍以及随机化

randomizationFactor

0.5

0 <= randomizationFactor <= 1

timeout

20000

在发出 connect_error 和 connect_timeout 事件之前的连接超时

autoConnect

true

通过将此设置为false,只要您认为合适,就必须调用 manager.open

query

{}

连接命名空间时发送的其他查询参数(然后在服务器端的 socket.handshake.quer y 对象中找到)

parser

要使用的解析器.默认为socket.io附带的 Parser 的实例.请参阅 socket.io-parser.

底层Engine.IO客户端的可用选项:

选项

默认值

描述

upgrade

true

客户端是否应该尝试将传输从长轮询升级到更好的状态.

forceJSONP

false

强制JSONP进行轮询传输.

jsonp

true

确定是否在必要时使用JSONP进行轮询.如果禁用(通过设置为false),则会发出错误(如果没有其他传输可用,则说“没有传输可用”).如果另一个传输可用于打开连接(例如WebSocket),则将使用该传输.

forceBase 64

false

即使XHR2 responseType可用,也强制基本64编码用于轮询传输,即使使用的标准支持二进制,也使用WebSocket.

enablesXDR

false

为IE8启用XDomainRequest,以避免因点击声音加载条形闪烁.默认为 false,因为XDomainRequest有一个不发送cookie的缺陷.

timestampRequests

是否为每个传输请求添加时间戳.注意：除非将此选项显式设置为 false,否则始终会标记轮询请求

timestampParam

t

时间戳参数

policyPort

843

策略服务器侦听的端口

transports

['polling', 'websocket']

要尝试的传输列表(按顺序). Engine 总是试图直接与第一个连接,前提是它的特征检测测试通过.

transportOptions

{}

选项的哈希值,由传输名称索引,覆盖给定传输的公共选项

rememberUpgrade

false

如果为true,并且先前与服务器的websocket连接成功,则连接尝试将绕过正常的升级过程并最初尝试websocket.传输错误后的连接尝试将使用正常的升级过程.建议您仅在使用SSL/TLS连接时打开此选项,或者如果您知道网络未阻止websockets.

onlyBinaryUpgrades

false

运输升级是否应限于支持二进制数据的运输

requestTimeout

0

xhr-polling请求的超时(以毫秒为单位)(0)(仅用于轮询传输)

protocols

子协议列表(参见`MDN参考 <https://developer.mozilla.org/en-US/docs/Web/API/WebSockets_API/Writing_WebSocket_servers #Subprotocols>`_)(仅适用于websocket传输)

底层Engine.IO客户端的Node.js-only选项：

选项

默认值

描述

agent

false

http.Agent 使用

pfx

用于SSL的证书,私钥和CA证书.

key

用于SSL的私钥.

passphrase

私钥或pfx的密码短语.

cert

公共x509证书使用.

ca

用于检查远程主机的授权证书或授权证书数组.

ciphers

描述要使用或排除的密码的字符串.有关格式的详细信息,请参阅 cipher格式列表.

rejectUnauthorized

false

如果为true,则根据提供的CA列表验证服务器证书.如果验证失败,则会发出“错误”事件.在发送HTTP请求之前,验证发生在连接级别.

perMessageDeflate

true

WebSocket permessage-deflate扩展的参数(参见 ws module api docs).设置为“false”以禁用.

extraHeaders

{}

将每个请求传递给服务器的标头(通过xhr-polling和websockets).然后可以在握手期间或特殊代理中使用这些值.

forceNode

false

对websockets使用NodeJS实现 - 即使有可用的本机Browser-Websocket,默认情况下优先于NodeJS实现. (这在使用nw.js或者电子等混合平台时非常有用)

localAddress

要连接的本地IP地址
